# Story 5.1: Schema Supabase — Products + Listings + Analyses

**Epic:** Produto Base com Listings por Canal (Epic 5)
**ID:** STORY-5.1
**Status:** Done
**Complexity:** S (3 points)
**Time Estimate:** 1 day

---

## Story

Como desenvolvedor, preciso criar o schema no Supabase para suportar o modelo de dados:
**produto base** (1) → **listings por canal** (N) → **histórico de análises** (N),
de forma que cada produto possa existir em múltiplos marketplaces com análises independentes.

**Dependencies:** None

---

## Acceptance Criteria

1. Tabela `products` criada com campos: id, name, description, category, brand, created_by, created_at, updated_at
2. Tabela `product_listings` criada com: id, product_id (FK), marketplace, url, title, price, image_url, listing_score, status, created_at, updated_at
3. Tabela `listing_analyses` criada com: id, listing_id (FK), score, summary, strengths (jsonb), weaknesses (jsonb), analysis_data (jsonb), analyzed_at
4. Coluna `listing_id` adicionada na tabela `tasks` existente (nullable, FK para product_listings)
5. RLS habilitado em todas as tabelas — usuário só vê seus próprios produtos
6. Índices criados: `product_listings.product_id`, `product_listings.marketplace`, `listing_analyses.listing_id`
7. Migration SQL salva em `supabase/migrations/`
8. Types TypeScript gerados/atualizados em `lib/types/products.ts`
