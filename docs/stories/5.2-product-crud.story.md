# Story 5.2: CRUD de Produto Base — Cadastro e Listagem

**Epic:** Produto Base com Listings por Canal (Epic 5)
**ID:** STORY-5.2
**Status:** Ready
**Complexity:** M (5 points)
**Time Estimate:** 2 days

---

## Story

Como usuário, quero cadastrar produtos base (nome, categoria, marca, descrição) e ver todos os meus produtos em uma lista, para organizar meu catálogo antes de adicionar os listings por canal.

**Dependencies:** STORY-5.1 (schema criado)

---

## Acceptance Criteria

1. Formulário de cadastro de produto com campos: nome (obrigatório), categoria, marca, descrição
2. Produto salvo na tabela `products` no Supabase com `created_by` = usuário logado
3. Listagem de produtos mostrando: nome, categoria, quantidade de listings, score médio entre canais
4. Botão "Gerenciar listings" em cada produto leva para a tela de listings (Story 5.3)
5. Editar e deletar produto disponíveis (delete remove listings e analyses associados via CASCADE)
6. Estado vazio amigável quando não há produtos
7. Loading states em todas as operações assíncronas
8. Rota: `/produtos` (substituir a página existente)

---

## Scope

### IN
- Página `/produtos` refatorada (remover lógica de URL scraping atual)
- API routes: `GET/POST /api/products`, `PUT/DELETE /api/products/[id]`
- Formulário inline ou modal para criação/edição
- Lista com cards de produto

### OUT
- Upload de imagem do produto base (feito no listing, Story 5.3)
- Paginação (implementar quando houver >20 produtos)
- Filtros e busca (P2)
- Import de planilha para produto base (Story 5.4)
