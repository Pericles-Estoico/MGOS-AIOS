# Story 5.4: Import em Lote — Planilha CSV/Excel e Documento

**Epic:** Produto Base com Listings por Canal (Epic 5)
**ID:** STORY-5.4
**Status:** Ready
**Complexity:** M (8 points)
**Time Estimate:** 2-3 days

---

## Story

Como usuário, quero importar múltiplos produtos e seus listings de uma vez via planilha
(CSV ou Excel .xlsx) ou documento (PDF/DOCX), para popular o sistema rapidamente
sem precisar cadastrar um a um.

**Dependencies:** STORY-5.1, STORY-5.2, STORY-5.3

---

## Acceptance Criteria

1. Página de import acessível via botão "Importar" na listagem de produtos
2. Suporte a upload de: `.csv`, `.xlsx`, `.pdf`, `.docx`
3. Template CSV/Excel disponível para download com colunas esperadas
4. Preview dos dados antes de confirmar import (primeiras 5 linhas)
5. Colunas esperadas na planilha: `nome`, `categoria`, `marca`, `marketplace`, `url`, `preco` (url e preco opcionais)
6. Linhas com `url` preenchida disparam análise AI automática após import (mesma lógica da Story 5.3)
7. Relatório de resultado: X produtos criados, X listings criados, X erros (com detalhes do erro por linha)
8. Documento PDF/DOCX: texto extraído e enviado para AI identificar produtos e criar cards para revisão humana antes de salvar
9. Import de até 50 produtos por vez (limite inicial)
10. Erros em linhas individuais não travam o import inteiro

---

## Scope

### IN
- Página `/produtos/import`
- Upload e parse de CSV (nativo) e Excel .xlsx (lib `xlsx`)
- Upload e extração de texto de PDF (`pdf-parse`) e DOCX (`mammoth`)
- Preview de dados antes de confirmar
- Template CSV para download
- API de import com tratamento de erros por linha
- Disparo de análise AI para listings com URL

### OUT
- Import de mais de 50 itens por vez (implementar com BullMQ nas Stories 4.x)
- Google Sheets integration
- Import agendado / recorrente
- Deduplicação automática de produtos com mesmo nome
